5.11 实现了mcp架构，由于是本机开发的demo，用官方的mcp python sdk并通过stdio通信。
stdio：
MCP Server 和 MCP Client 运行在同一个进程内，通过标准输入（stdin）和标准输出（stdout） 来读写消息。
数据流向：客户端往 stdin 写请求，服务器从 stdin 读取；服务器往 stdout 写响应，客户端从 stdout 读取。
消息格式：每一条 JSON-RPC 消息前后各加一个 \n 换行符，成为一行一行的纯文本流。
| 优点 | 缺点 |
|------|------|
| **零配置**：无需网络端口，无需防火墙，无需认证。 | **只能本地使用**：Client 和 Server 必须在同一台机器，同一进程。 |
| **性能极好**：纯内存管道，无网络延迟，低开销。 | **无法通过 HTTP 暴露**：你无法让远程客户端调用该 Server。 |
| **调试方便**：可以直接在终端跑 `python server.py` 查看日志。 | **一个 Server 只能服务一个 Client**（因为 stdin/stdout 是单向并独占的）。 |
